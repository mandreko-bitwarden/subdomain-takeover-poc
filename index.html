What is Subdomain Takeover?

Subdomain takeover is a security vulnerability that occurs when a subdomain (e.g., blog.example.com) points to an external service (like GitHub Pages, AWS S3, or other cloud services) that is no longer in use or has been improperly configured.

An attacker can claim the unconfigured resource and serve malicious content on the legitimate subdomain, potentially:

  • Hosting phishing pages that appear to come from the legitimate domain
  • Distributing malware under a trusted domain name
  • Damaging the organization's reputation
  • Stealing cookies and sensitive information from users
  • Bypassing security controls that trust the parent domain

How to Prevent Subdomain Takeover

  • Regularly audit DNS records and remove entries pointing to decommissioned services
  • Monitor for dangling DNS records that point to unclaimed resources
  • Verify ownership of external services before creating DNS records
  • Use tools to scan for subdomain takeover vulnerabilities
  • Implement proper change management when decommissioning services
